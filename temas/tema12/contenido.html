Aplicaciones de la Lógica de Predicados

Introducción a la Programación Lógica y Prolog

Progreso del Tema 0%

Introducción y Objetivos

En este tema final del curso, exploraremos cómo los conceptos de lógica de predicados que hemos estudiado se pueden aplicar de manera práctica en la programación lógica. Nos enfocaremos en Prolog, el lenguaje de programación lógica más conocido y utilizado.

Objetivos del Tema

  • Comprender la relación entre lógica de predicados y programación lógica
  • Conocer los elementos fundamentales de Prolog
  • Aprender a escribir hechos, reglas y consultas básicas
  • Entender el concepto de corte en programación lógica
  • Explorar aplicaciones prácticas de la programación lógica

Presentación Informal de la Programación Lógica

La programación lógica es un paradigma de programación que se basa en la lógica formal. Prolog (Programming in Logic) es el lenguaje más representativo de este paradigma y constituye un intento de automatización de la lógica de predicados.

Concepto Fundamental

La idea general consiste en poder expresar conocimiento en forma de programas con un lenguaje semejante al de la lógica de predicados, de manera que un computador sea capaz de procesar ese programa y que un usuario pueda hacer consultas sobre ese conocimiento previamente establecido.

Estructura de un Programa Prolog

Un programa Prolog consta de dos partes fundamentales:

  • Base de hechos: Conocimiento básico sobre el dominio
  • Base de reglas: Conocimiento derivado mediante reglas lógicas

Elementos del Prolog

Los Predicados

Los predicados en Prolog se expresan con un functor y una serie de términos entre paréntesis y separados por comas. Para un predicado n-ádico, la forma es:

functor(termino1, termino2, ..., terminon)

Los Hechos y la Base de Hechos

Los hechos se expresan mediante functores y los términos mediante constantes de individuo, que en Prolog se expresan en minúsculas.

Ejemplo de Base de Hechos

% Animales cánidos
canido(perro).
canido(lobo).

% Animales felinos
felino(gato).
felino(leon).
felino(tigre).

% Plantas
arbol(roble).
arbol(pino).
arbol(encina).

El Diálogo

A partir de un programa Prolog, podemos realizar consultas. El sistema responde con true o false para consultas con constantes, y con los valores posibles para consultas con variables.

Ejemplos de Consultas

% Consultas con constantes
?- canido(perro).
true.

?- felino(perro).
false.

% Consultas con variables
?- canido(X).
X = perro ;
X = lobo.

Las Variables

Las variables en Prolog se representan en mayúsculas y permiten hacer consultas más interesantes y generales.

Las Reglas

Las reglas añaden mayor potencialidad al programa. Permiten definir conceptos derivados a partir de los hechos básicos.

Ejemplo de Reglas

% Un mamífero es un felino o un cánido
mamifero(X) :- felino(X).
mamifero(X) :- canido(X).

% Una planta es un árbol
planta(X) :- arbol(X).

Caracteres y Constantes

Los Caracteres

Prolog soporta diferentes tipos de caracteres, aunque puede haber variaciones entre diferentes intérpretes. Es importante consultar la documentación específica del intérprete que se esté utilizando.

Las Constantes

En Prolog existen dos tipos principales de constantes:

Tipos de Constantes

  • Constantes atómicas: Se escriben en minúsculas sin tildes ni espacios en blanco
  • Constantes numéricas: Números enteros y reales
% Constantes atómicas
perro, gato, arbol

% Constantes numéricas
123, 3.14, -5

Seguimiento o Traza de un Programa

Una de las ventajas de Prolog es que, ante cualquier pregunta, podemos seguir paso a paso todo el proceso que el sistema sigue para llegar a la respuesta. A esto se le denomina traza y es un procedimiento altamente interesante y de gran ayuda cuando estamos programando en Prolog.

Ventajas de la Traza

  • Permite entender cómo Prolog resuelve las consultas
  • Facilita la depuración de programas
  • Ayuda a comprender el mecanismo de unificación
  • Es útil para optimizar el rendimiento del programa

Unificación: El Corazón de Prolog

La unificación es el proceso mediante el cual Prolog hace coincidir términos. Es el mecanismo fundamental que permite que Prolog funcione y es esencial para entender cómo se resuelven las consultas.

¿Qué es la Unificación?

La unificación es el proceso de hacer que dos términos sean idénticos mediante la sustitución de variables. Dos términos se unifican si:

  • Son idénticos
  • Pueden hacerse idénticos sustituyendo variables

Ejemplos de Unificación

% Unificación exitosa
padre(juan, maria) unifica con padre(X, Y)
% Resultado: X = juan, Y = maria

% Unificación con listas
[1, 2, 3] unifica con [X | Y]
% Resultado: X = 1, Y = [2, 3]

Reglas de Unificación

  • Constantes: Solo se unifican consigo mismas
  • Variables: Se unifican con cualquier término
  • Estructuras: Se unifican si tienen el mismo functor y aridad

Backtracking: La Búsqueda de Soluciones

El backtracking es el mecanismo que permite a Prolog encontrar múltiples soluciones. Cuando una consulta falla, Prolog "vuelve atrás" y prueba alternativas.

¿Cómo Funciona el Backtracking?

Prolog mantiene un "punto de elección" para cada regla. Cuando una consulta falla:

  1. Vuelve al último punto de elección
  2. Prueba la siguiente alternativa
  3. Continúa la búsqueda

Ejemplo de Backtracking

% Base de hechos
animal(perro).
animal(gato).
animal(pajaro).

% Consulta: ?- animal(X).
% Prolog encuentra: X = perro
% Al presionar ';' busca más: X = gato
% Y luego: X = pajaro

Puntos Clave del Backtracking

  • Exhaustivo: Prolog busca todas las soluciones posibles
  • Sistemático: Sigue un orden determinado
  • Eficiente: Solo explora ramas relevantes

Listas en Prolog

Las listas son estructuras de datos fundamentales en Prolog. Permiten manejar colecciones de elementos de manera eficiente.

Sintaxis de Listas

Formas de Representar Listas

% Lista completa
[1, 2, 3, 4]

% Lista con cabeza y cola
[Primero | Resto]

% Lista vacía
[]

Predicados Útiles con Listas

Predicados Básicos

% Verificar si un elemento está en la lista
member(X, [X | _]).
member(X, [_ | T]) :- member(X, T).

% Concatenar dos listas
append([], L, L).
append([X | L1], L2, [X | L3]) :-
    append(L1, L2, L3).

Recursión en Prolog

La recursión es fundamental en Prolog. Muchos predicados se definen de manera recursiva, donde un caso base termina la recursión y un caso recursivo reduce el problema.

Estructura de la Recursión

  • Caso Base: Condición de parada
  • Caso Recursivo: Llamada recursiva con problema más pequeño

Ejemplo: Factorial Recursivo

% Caso base: factorial de 0 es 1
factorial(0, 1).

% Caso recursivo
factorial(N, F) :-
    N > 0,
    N1 is N - 1,
    factorial(N1, F1),
    F is N * F1.

Recursión con Listas

Longitud de una Lista

% Caso base: lista vacía tiene longitud 0
longitud([], 0).

% Caso recursivo
longitud([_ | T], L) :-
    longitud(T, L1),
    L is L1 + 1.

La Reevaluación y el Corte

Cuando hacemos consultas en Prolog, el intérprete es completamente exhaustivo y va a proporcionarnos siempre todas las respuestas posibles. En ocasiones no resulta necesario que el programa proporcione todas y cada una de las respuestas.

Este es el motivo por el cual Prolog incorpora un procedimiento para evitar que las respuestas sean exhaustivas: el corte (representado por !).

Ejemplo del Corte

% Sin corte - devuelve todas las respuestas
mamifero(X) :- felino(X).
mamifero(X) :- canido(X).

% Con corte - se detiene en la primera respuesta
mamifero(X) :- felino(X), !.
mamifero(X) :- canido(X).
